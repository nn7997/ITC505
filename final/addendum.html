Addendum: Detailed Description of LIGHTS OUT Game Development

Objective:

The Lights Out game is a grid-based puzzle where the player aims to turn off all the lights on the board. The game is implemented using HTML, CSS, and JavaScript to create an interactive and visually appealing user experience.

HTML Structure:

  1. Game Title: The game title is displayed prominently using an <h1> element, providing a clear indication of the game's identity.
  2. Game Board: The game board is represented as a grid using the <div> element with the class board. Each square is represented as a child <div> within the board container.
  3. Buttons and Metrics: A "New Game" button triggers the start of a new game. The number of moves and a timer indicating the elapsed time are displayed using <p> elements.

CSS Styling:

  1. Background: The body has a background image (GIF) covering the entire viewport, creating an immersive ambiance for the game.
  2. Container Styling: The game container is centered vertically and horizontally using flexbox, ensuring a responsive layout.
  3. Square Styling: Each square on the game board is styled with a fixed size, background image (representing a light), and borders. The is-off class toggles the appearance of each square to create the effect of turning off the lights.
  4. Button Styling: New Game button and other buttons have a distinct style, using transitions to enhance the interactive feel.

JavaScript Logic:

  1. Moves and Move Limit: The moves variable tracks the number of moves made by the player. A maximum move limit (MAX_MOVES) is set to provide a challenge. The move count is displayed to the user.
  2. Timer Functionality: The game features a timer that starts when the game begins and stops upon winning or reaching the move limit. The timer displays elapsed time in minutes and seconds.
  3. Square Toggling: Clicking on a square toggles its state and the state of adjacent squares, creating a chain reaction effect.
  4. Game State Checking: The game checks for a win condition when all squares are turned off (isWin). Additionally, a loss condition is triggered if the move limit is reached.
  5. Alerts on Win/Loss: When the game is won or lost, a congratulatory or game-over message is displayed, including the number of moves and time taken.

Game Flow:

  1. Initialization: The game initializes with a new game, setting moves and timer to zero, and starting the timer.
  2. Square Interaction: Clicking on a square triggers the toggling of its state and adjacent squares.
  3. Game State Check: After each move, the game checks whether the win or loss conditions are met.
  4. End Game Alerts: If the player wins, a congratulatory alert is shown with the number of moves and time taken. If the player loses, a game-over alert is displayed.

Conclusion:

The Lights Out game provides an engaging and challenging puzzle experience, encouraging players to strategize and efficiently turn off all the lights within the move limit. The combination of visual appeal, interactive design, and logic implementation makes this game an enjoyable and rewarding experience for users.
